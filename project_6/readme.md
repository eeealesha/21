### Задание 1: CRISP-DM и постановка задачи

**Для выполнения нужно знать:** методология CRISP-DM, перевод бизнес-вопроса в аналитическую задачу, KPI, data-driven инфраструктура.

Любой аналитический проект начинается не с кода, а с понимания бизнес-контекста. CRISP-DM — это стандартный framework, который помогает не потеряться на пути от вопроса к ответу.

### Задание 3: Подготовка данных

**Для выполнения нужно знать:** pandas merge, feature engineering, EDA, корреляции.

Сырые данные — это записи о визитах. Для анализа оттока нужны данные на уровне пациента. Этот шаг — превращение строк визитов в профиль каждого пациента.

**Выполни следующие шаги:**

1. Загрузи `patients.csv` и `visits.csv`. Объедини через `pd.merge(how='left')` по `patient_id`.
2. Создай аналитическую таблицу на уровне пациента через `groupby('patient_id').agg(...)`. Рассчитай для каждого пациента:
   - `total_visits` — всего визитов
   - `conducted_visits` — проведённых визитов
   - `cancel_rate` — доля отменённых (`Отменён` / total)
   - `avg_revenue` — средняя выручка (только `«Проведён»`)
   - `avg_wait_time` — среднее время ожидания (только `«Проведён»`)
   - `days_since_last_visit` — дней между последним визитом и 2024-12-31
   - `churned` — 1 если `days_since_last_visit > 180`, иначе 0
   - `tenure_months` — стаж в клинике от `registration_date` до последнего визита в месяцах
3. Объедини с демографическими данными из `patients.csv` (age, gender, insurance_type, district).
4. Проведи EDA: распределения числовых признаков (`sns.histplot`), `sns.countplot` churned, heatmap корреляций признаков с `churned`.
5. Markdown-вывод: перечисли топ-3 признака, которые наиболее коррелируют с `churned`.

### Задание 4: Анализ факторов оттока

**Для выполнения нужно знать:** когортный анализ, логистическая регрессия, sklearn, ROC AUC, интерпретация коэффициентов.

Корреляция — это подсказка. Модель — это ответ. Логистическая регрессия предскажет вероятность оттока для каждого пациента и покажет, какие факторы важнее всего.

**Выполни следующие шаги:**

1. **Когортный анализ**: сгруппируй пациентов по кварталу первого визита (Q1 2022, Q2 2022, ...). Для каждой когорты рассчитай Retention Rate через 90, 180 и 365 дней после первого визита. Построй `sns.lineplot` — ось X: когорта, ось Y: retention rate, отдельные линии для каждого горизонта (90/180/365 дней).

2. **Анализ по группам**: для каждого из следующих факторов — построй `sns.boxplot` или `sns.barplot` с разбивкой churned/retained и напиши Markdown-вывод:
   - Возрастная группа (18–30, 31–45, 46–60, 60+)
   - Тип страхования (`insurance_type`)
   - Среднее время ожидания (`avg_wait_time`)
   - Доля отменённых (`cancel_rate`)

3. **Логистическая регрессия**:
   - Признаки: `age`, `cancel_rate`, `avg_wait_time`, `avg_revenue`, `total_visits`, `tenure_months` + dummy-переменные для `insurance_type` (`pd.get_dummies`)
   - Разбить train/test (80/20, `random_state=42`)
   - Масштабировать через `StandardScaler`
   - Обучить `LogisticRegression(max_iter=1000, random_state=42)`
   - Вывести: `accuracy_score`, `roc_auc_score`, `classification_report`
   - Построить ROC-кривую: `plt.plot(fpr, tpr, label=f'AUC = {auc:.2f}')`

4. Вывести топ-3 признака по `abs(coef_)`. Markdown-вывод: что означает каждый коэффициент для клиники?
