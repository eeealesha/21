Когда ты проверяешь чужой дашборд, ты видишь другой подход к той же задаче: другую структуру layout, другой способ написать callback, другую цветовую схему. Это не значит, что один из вас ошибся — это значит, что у задачи несколько правильных решений.

## Общие правила проверки

- На проверку принимаются только файлы, которые находятся в папке `src/` ветки `develop` репозитория проверяемого участника.
- Убедись, что ты открыл репозиторий именно того человека, которого должен проверить.
- Перечитай задание перед проверкой каждого пункта.
- **Перед запуском каждого `.py` файла**: убедись что зависимости установлены (`pip install dash pandas plotly`). Запусти файл и проверь, что приложение открывается в браузере без ошибок.
- **Перед запуском каждого `.ipynb` файла**: выполни Kernel → Restart & Run All. Ноутбук должен запуститься без ошибок от первой до последней ячейки.
- **EMPTY WORK** — поставь этот флаг, если репозиторий пустой, ветка `develop` отсутствует, или файлы задания находятся не в папке `src/`. Продолжи проверку, чтобы помочь разобраться.
- **CHEAT** — поставь этот флаг, если участник не может объяснить логику своего кода, не понимает что делает каждая строка, или решения явно идентичны чужим без понимания. Продолжи проверку.
