## Chapter II — Задания

Финансовый директор «Здоровье+» устал от Excel-таблиц. Ему нужен живой дашборд, который обновляется при выборе нужного врача или периода — без перезагрузки страницы, без ручной фильтрации, без звонков аналитику.

Твоя задача — пройти путь от первого `px.bar()` до полноценного веб-приложения, которое директор сможет открыть в браузере и самостоятельно исследовать данные клиники.

**Файлы с данными:** `datasets/clinic_visits.csv` и `datasets/clinic_staff.csv`.

`clinic_visits.csv` содержит 498 записей о визитах пациентов за 2024 год:

| Столбец | Описание |
|---------|----------|
| `visit_id` | Идентификатор визита |
| `visit_date` | Дата визита (YYYY-MM-DD) |
| `patient_id` | Идентификатор пациента |
| `patient_name` | ФИО пациента |
| `age` | Возраст пациента (есть пропуски) |
| `gender` | Пол: «М» / «Ж» |
| `district` | Район города (есть пропуски) |
| `doctor_id` | Идентификатор врача |
| `doctor_name` | ФИО врача |
| `specialty` | Направление: Терапия, Хирургия, Неврология, Кардиология, Педиатрия, Стоматология |
| `branch` | Филиал клиники |
| `status` | Статус: `«Проведён»` / `«Отменён»` |
| `revenue` | Выручка (₽); 0 для отменённых визитов |
| `wait_time_min` | Время ожидания (мин); 0 для отменённых |

`clinic_staff.csv` содержит данные о 24 врачах: `doctor_id`, `doctor_name`, `specialty`, `branch`, `hire_date`, `salary_per_visit`.

### Задание 1: Панорама инструментов и первые графики

**Для выполнения нужно знать:** типы BI-инструментов, библиотека `plotly.express`, Jupyter Notebook.

Сохрани в файл `task1_bi_overview.md`.

### Задание 3: Знакомство с Dash

**Для выполнения нужно знать:** PyCharm, `dash`, `dcc`, `html`, декоратор `@app.callback`, `Input`, `Output`.

Графики в Jupyter — это хорошо. Но они статичные: чтобы посмотреть другое направление, нужно перезапускать ячейку. Dash решает эту проблему: он превращает Python-скрипт в веб-приложение с живыми элементами управления.

**Выполни следующие шаги:**

1. Установи зависимости: `pip install dash pandas plotly`.
2. Создай проект в PyCharm. Изучи шаблон из файла `materials/task3_example.py` — он показывает только формат, не решение.
3. Создай файл `task3_app.py`. Напиши Dash-приложение:
   - Загрузи `clinic_visits.csv` через pandas.
   - Добавь `html.H1` с заголовком «Клиника Здоровье+».
   - Добавь `dcc.Dropdown` для выбора одного направления (`specialty`). Значение по умолчанию — первое в списке.
   - Добавь `dcc.Graph` — bar chart выручки по месяцам для выбранного направления (только `«Проведён»`).
   - Напиши `@app.callback`: при изменении Dropdown пересчитывает и возвращает обновлённый график.
4. Запусти приложение: `python task3_app.py`. Открой `http://127.0.0.1:8050/` в браузере. Убедись, что выбор направления меняет график.
5. Сделай скриншот работающего приложения: `task3_screenshot.png`.

**Результат:** файлы `task3_app.py` и `task3_screenshot.png` в папке `src/`.

### Задание 5: Многостраничный дашборд

**Для выполнения нужно знать:** `dcc.Tabs`, `pd.merge`, `dcc.Store`, `dash_table.DataTable`.

Один экран — одна история. Но у клиники их несколько: финансовая и кадровая. Пора разложить всё по вкладкам.

**Выполни следующие шаги:**

1. Создай файл `task5_app.py`. Объедини `clinic_visits.csv` и `clinic_staff.csv` через `pd.merge(how='left')` по `doctor_id`.
2. Добавь `dcc.Dropdown` по `branch` (над вкладками) — глобальный фильтр по филиалу.
3. Создай `dcc.Store(id='filtered-data')` для хранения отфильтрованных данных. Первый callback: по значению Dropdown → записывает JSON в `dcc.Store`.
4. Создай `dcc.Tabs` с двумя вкладками:
   - **«Финансы»**: line chart динамики выручки по месяцам + scatter `wait_time_min` vs `revenue`. Оба графика читают данные из `dcc.Store`.
   - **«Врачи»**: bar chart топ-5 врачей по суммарной выручке + `dash_table.DataTable` с их данными (имя, направление, филиал, стаж в годах).
5. Стаж в годах (`tenure_years`) рассчитай через `hire_date`: `(pd.Timestamp.today() - pd.to_datetime(df['hire_date'])).dt.days // 365`.
6. Сделай по одному скриншоту каждой вкладки: `task5_screenshot_finance.png` и `task5_screenshot_doctors.png`.

**Результат:** файлы `task5_app.py`, `task5_screenshot_finance.png`, `task5_screenshot_doctors.png` в папке `src/`.
