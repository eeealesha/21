# Checklist: Проект 2 — Работа с базами данных и SQL

## Введение

Проверка — это не формальность. Это ключевая часть обучения.

Когда ты проверяешь чужое решение, ты видишь другой подход к той же задаче. Может быть, более элегантный. Может быть, с ошибкой, которую ты сам чуть не совершил. Может быть, с идеей, которая изменит твоё собственное понимание SQL.

Не ищи ошибки ради ошибок. Ищи понимание. Задавай вопросы. Обсуждай решения. Именно в этом разговоре — настоящее обучение.

---

## Общие правила проверки

- На проверку принимаются только файлы, которые находятся в папке `src/` ветки `develop` репозитория проверяемого участника.
- Убедись, что ты открыл репозиторий именно того человека, которого должен проверить.
- Перечитай задание перед проверкой каждого пункта.
- **EMPTY WORK** — поставь этот флаг, если репозиторий пустой, ветка `develop` отсутствует, или файлы задания находятся не в папке `src/` ветки `develop`. Продолжи проверку, чтобы помочь разобраться.
- **CHEAT** — поставь этот флаг, если участник не может объяснить логику своих запросов, не знает, что возвращает каждый из них, или решения явно идентичны чужим без понимания. Продолжи проверку.
