Проект 2: Погружение в SQL и Реляционные базы данных

**Аннотация:** Данный проект направлен на изучение языка SQL — главного инструмента работы с данными. Тебе предстоит погрузиться в архитектуру реляционных баз данных на примере классической базы сети видеопрокатов (DVD Rental). Ты пройдешь путь от простых выборок до многоуровневых аналитических запросов с использованием оконных функций. В финале проекта ты научишься не просто писать код, а собирать разрозненные таблицы в единые датасеты, способные отвечать на сложные вопросы бизнеса.

- Взаимообучение (P2P) — это процесс, при котором учащиеся обмениваются знаниями. Читая чужой код (SQL-запросы), ты учишься оптимизации.

## Chapter II
### Вводная информация (Реляционные БД и SQL)

Ты устроился дата-аналитиком в сеть ретро-видеопрокатов. База данных компании — это типичная реляционная модель: информация об актерах (`actor`), фильмах (`film`), платежах (`payment`), инвентаре (`inventory`) и клиентах (`customer`) хранится в 15 разных таблицах. Они связаны между собой ключами (Primary Key и Foreign Key). 

Чтобы руководство могло принимать решения об акциях, закупках новых фильмов и премировании сотрудников, тебе нужно научиться извлекать эти данные.

## Chapter III
Цель проекта — написать набор SQL-скриптов для решения реальных бизнес-задач компании, поэтапно применяя все конструкции языка.

### Задание 1. Первичный скрининг каталога
- **Что нужно знать?** `SELECT`, `FROM`, `WHERE`, `ORDER BY`, `DISTINCT`, логические операторы (`AND`, `OR`, `IN`).
- **Контекст:** Контент-менеджер хочет запустить акцию "Семейные выходные". Ему нужен список фильмов, подходящих для детей, но чтобы они не были слишком короткими.
- **Задача:** Выгрузить список уникальных названий фильмов из таблицы `film`.
- **Действие:** Напиши запрос, который выведет названия (`title`), год выпуска (`release_year`) и продолжительность (`length`). Оставь только те фильмы, рейтинг (`rating`) которых 'G' или 'PG', а продолжительность от 90 до 120 минут включительно. Отсортируй результат по продолжительности по убыванию. Дубликатов быть не должно.
- **Результат:** Файл `task1_basic.sql`.

### Задание 2. Агрегация смыслов
- **Что нужно знать?** Агрегатные функции (`COUNT`, `SUM`, `AVG`), `GROUP BY`, фильтрация групп `HAVING`.
- **Контекст:** Отдел лояльности планирует раздать VIP-карты самым активным клиентам. Нужно понять, кто приносит больше всего денег, но при этом учесть только проверенных пользователей.
- **Задача:** Найти самых прибыльных клиентов компании.
- **Действие:** Используя таблицу платежей (`payment`), сгруппируй данные по `customer_id`. Посчитай количество платежей и общую сумму (`amount`) для каждого клиента. Оставь (отфильтруй) только тех клиентов, которые совершили более 30 платежей за всё время. Отсортируй результат по общей сумме по убыванию.
- **Результат:** Файл `task2_aggregation.sql`.

### Задание 3. Паутина связей
- **Что нужно знать?** Типы соединений (`INNER JOIN`, `LEFT JOIN`), алиасы таблиц (`AS`).
- **Контекст:** Маркетологи хотят понять жанровые предпочтения. Проблема в том, что названия фильмов лежат в одной таблице, категории — в другой, а связь между ними — в третьей (`film_category`).
- **Задача:** Собрать единую витрину данных по фильмам жанра 'Comedy' и 'Action'.
- **Действие:** Напиши запрос, который соединяет таблицы `film`, `film_category` и `category`. Выведи название фильма, описание и название категории. Используй `INNER JOIN`. 
Вторым запросом в этом же файле используй `LEFT JOIN` между таблицами `film` и `inventory`, чтобы найти названия фильмов, которых вообще нет в наличии (ни в одном магазине).
- **Результат:** Файл `task3_joins.sql` (должен содержать 2 запроса, разделенных точкой с запятой).

### Задание 4. Матрешка данных
- **Что нужно знать?** Вложенные подзапросы (Subqueries) в конструкциях `WHERE` и `SELECT`.
- **Контекст:** Финансовый директор попросил найти "китов" — клиентов, чей средний чек за аренду значительно превышает средний чек по всей сети.
- **Задача:** Использовать подзапрос для динамического вычисления среднего значения и сравнения с ним.
- **Действие:** Напиши запрос к таблице `payment`. Выведи `customer_id` и средний платеж этого клиента. С помощью подзапроса в `WHERE` (или `HAVING`) сделай так, чтобы в итоговую выборку попали только те клиенты, чей средний платеж строго больше, чем средний платеж по всей таблице `payment` среди всех клиентов.
- **Результат:** Файл `task4_subqueries.sql`.

### Задание 5. Властелин времени и условий
- **Что нужно знать?** Функции работы с датами (в PostgreSQL — `EXTRACT` или `TO_CHAR`), условный оператор `CASE WHEN`.
- **Контекст:** Аналитик строит дашборд. Ему нужна новая категориальная переменная для оценки длительности фильмов и анализ выручки по дням недели.
- **Задача:** Разметить данные с помощью условий и извлечь день недели из даты платежа.
- **Действие:** Напиши запрос к таблицам `film` и `payment`. 
1. Для фильмов: выведи `title` и создай новый столбец `duration_category`. Если фильм короче 60 минут — 'Short', от 60 до 120 — 'Standard', более 120 — 'Long' (используй `CASE`).
2. Для платежей: извлеки из `payment_date` день недели и просуммируй выручку. Выведи две колонки: день недели и общая сумма.
- **Результат:** Файл `task5_conditions.sql` (2 запроса).

### Задание 6. Окна в аналитику
- **Что нужно знать?** Оконные функции (`OVER`, `PARTITION BY`, `ORDER BY`), `ROW_NUMBER()`, `LAG()`.
- **Контекст:** Директор магазина хочет наградить Топ-3 клиентов каждого отдельного магазина (стора), а финдиректор — увидеть динамику выручки месяц к месяцу (MoM).
- **Задача:** Использовать оконные функции для ранжирования внутри групп и доступа к предыдущим строкам.
- **Действие:** 1. Соедини `customer`, `payment` и `store`. Посчитай сумму платежей каждого клиента. С помощью оконной функции `ROW_NUMBER()` или `RANK()` присвой им места внутри каждого магазина (`PARTITION BY store_id`). Оставь только Топ-3 для каждого магазина (подсказка: понадобится подзапрос или CTE).
2. Сгруппируй выручку (`payment`) по месяцам. С помощью функции `LAG()` выведи в одной строке выручку текущего месяца и выручку предыдущего месяца.
- **Результат:** Файл `task6_windows.sql` (2 запроса).
