# Аналитика данных: Курс «Здоровье+»

**Один год в жизни аналитика. Одна клиника. Шесть инструментов.**

Ты — аналитик данных в сети многопрофильных клиник «Здоровье+» (10 филиалов, 200+ врачей, 50 000 пациентов в год). Данные есть, но хаотичные. Руководство хочет решений, а не отчётов. Впереди — год работы, шесть проектов, шесть инструментов. Один и тот же вопрос: как превратить сырые данные в решение?

## Персонажи клиники

Во всех проектах действуют одни и те же люди. Знай их — общаешься с ними как аналитик.

| Роль | Контекст в курсе |
|------|-----------------|
| **Генеральный директор** | Заказчик стратегических решений. Хочет видеть картину целиком — без формул и p-value. Появляется в P1 (постановка задачи) и P6 (финальная стратегия). |
| **Финансовый директор** | Спрашивает про выручку, маржу, ROI. Не доволен ручными отчётами. Двигатель P1, P3, P4. |
| **Маркетолог** | Предложил SMS-напоминания. Инициатор A/B теста в P5. Думает про удержание пациентов в P6. |
| **CTO / IT-директор** | Предложил перейти с Excel на реляционную БД. Мост P1 → P2. |
| **Главврач** | Следит за загрузкой врачей и временем ожидания. Появляется в P3 и P4. |
| **Аналитик (ты)** | Карьерная дуга: «младший аналитик» (P1) → «аналитик» (P3–P5) → «старший аналитик» (P6). |

## Единый словарь

Эти термины и значения неизменны во всех шести проектах.

**Специализации:** Терапия, Хирургия, Неврология, Кардиология, Педиатрия.
В P4 к ним добавляется Стоматология — после открытия нового кабинета в Q3 того же года.

**Статусы визитов** (две нормализованные формы, всегда с ё):
- `«Проведён»` — визит состоялся
- `«Отменён»` — визит отменён или пациент не пришёл

В сырых данных P1 статусы намеренно «замусорены» (проведен, ПРОВЕДЕН, Выполнен и др.) — задача студента их нормализовать.

**Ключевые метрики:**
- **No-show rate** = доля отменённых визитов от общего числа
- **Выручка** = сумма по проведённым визитам (₽)
- **Время ожидания** = время от записи до приёма (минуты)
- **Retention Rate** = доля вернувшихся пациентов за период

## Технические соглашения

- **Воспроизводимость:** все генераторы данных используют `random.seed(42)`.
- **Запуск:** `python3 datasets/create_datasets.py` (или аналог проекта).
- **Решения:** сдаются в папку `src/` ветки `develop`.
- **Структура каждого проекта:** `readme.md` + `check_list.md` + `datasets/` + `materials/<шаблон>`.
- **P1 vs P2+ имена столбцов:** P1 использует русские имена (`Выручка_руб`, `ФИО_врача`) — так данные выгружает старая МИС клиники. С P2 вся БД переходит на английский snake_case (`revenue`, `doctor_id`) — стандарт современных аналитических систем. Это сделано намеренно.

**Зависимости Python (устанавливать перед P3–P6):**
```
pip3 install pandas numpy matplotlib seaborn plotly dash scipy scikit-learn fpdf2
```

## Как проходить курс

1. **Последовательно** — каждый проект ссылается на контекст предыдущего.
2. **Самостоятельно** — ни один шаблон в `materials/` не содержит ответов.
3. **С пиром** — проверки P2P обязательны; чек-лист в `check_list.md` каждого проекта.
4. **Не пропуская заключения** — финальные абзацы проектов объясняют, зачем ты только что это сделал и что будет дальше.
