## Chapter II — Задания

Руководство клиники «Здоровье+» устало терять деньги на отменённых визитах. Врач не пришёл — время потеряно, кабинет простоял, деньги не заработаны. Команда маркетинга предложила идею: отправлять пациентам SMS-напоминание за 24 часа до приёма.

Идея звучит здраво, но «звучит здраво» — не аргумент для финансового директора. Нужны данные. Именно поэтому был запущен A/B тест: половина пациентов (группа B) получала SMS, другая половина (группа A) — нет. Шесть месяцев спустя в твоих руках оказались данные эксперимента.

Твоя задача — разобраться, работают ли SMS-напоминания, и дать руководству конкретную рекомендацию.

---

**Файлы с данными:** `datasets/ab_test_data.csv` и `datasets/aa_test_data.csv`.

`ab_test_data.csv` содержит 800 записей за январь–июнь 2024 года:

| Столбец | Описание |
|---------|----------|
| `visit_id` | Идентификатор визита |
| `patient_id` | Идентификатор пациента |
| `group` | Группа эксперимента: `'A'` (без SMS) / `'B'` (с SMS) |
| `visit_date` | Дата визита (YYYY-MM-DD) |
| `specialty` | Направление: Терапия, Хирургия, Неврология, Кардиология, Педиатрия |
| `age` | Возраст пациента (есть пропуски) |
| `district` | Район города (есть пропуски) |
| `status` | Статус визита: `«Проведён»` / `«Отменён»` |
| `revenue` | Выручка (₽); 0 для отменённых визитов |
| `is_repeat` | 1 — повторный визит, 0 — первичный |

`aa_test_data.csv` содержит 600 записей с группами `AA_1` и `AA_2` — два случайных разбиения одной предэкспериментальной выборки для проверки корректности рандомизации.

### Задание 1: Постановка эксперимента

**Для выполнения нужно знать:** нулевая и альтернативная гипотезы, целевая метрика, MDE, power analysis, `statsmodels`.

Прежде чем анализировать результаты — нужно понять, что именно мы проверяем и как мы поймём, что результат значим. Аналитик без гипотезы — это просто человек, который смотрит на числа.

**Выполни следующие шаги:**

1. Изучи шаблон из `materials/task1_template.md` — он показывает только формат, не содержание.
2. В файле `task1_hypothesis.md` сформулируй:
   - Целевую метрику: **No-show rate** = доля `«Отменён»` от общего числа визитов в группе. Запиши формулу.
   - H₀ и H₁ в формальном виде: «H₀: p_B − p_A = 0», «H₁: p_B < p_A» (нас интересует снижение).
   - Вторичные метрики: средняя выручка за проведённый визит, доля повторных обращений.
   - Потенциальные угрозы: эффект новизны, загрязнение групп (пациент переходит из A в B), сезонность.
3. В файле `task1.ipynb` рассчитай необходимый размер выборки:
   - Базовый No-show rate (до теста): `p_base = 0.25`
   - MDE: снижение на 8 п.п. (то есть `p_B = 0.17`)
   - `alpha = 0.05`, `power = 0.80`
   - Использовать: `from statsmodels.stats.proportion import proportion_effectsize` и `from statsmodels.stats.power import NormalIndPower`
4. Добавь Markdown-ячейку: «Хватает ли нам фактического размера выборки (400 на группу)?»

### Задание 3: AA-анализ

**Для выполнения нужно знать:** логика AA-теста, chi-square, t-test, интерпретация p-value.

Прежде чем верить результатам A/B теста, нужно убедиться: а вдруг сплит-система изначально распределяла пациентов неравномерно? Для этого существует AA-тест — мы проверяем два «контроля», которые не должны отличаться.

**Выполни следующие шаги:**

1. Загрузи `aa_test_data.csv`. Группы `AA_1` и `AA_2` — случайные разбиения одной предэкспериментальной выборки. Выведи размеры обеих групп.
2. Сравни **No-show rate** групп AA_1 и AA_2 через chi-square. Вывести χ², p-value.
3. Сравни **средний revenue** (только `«Проведён»`) через t-test. Вывести t-статистику, p-value.
4. Сравни **распределение age** через t-test.
5. Для каждого теста — Markdown-ячейка: принимаем или отвергаем H₀? Что это означает?
6. Финальный вывод: «Сплит-система корректна / некорректна, потому что...»

### Задание 5: Статистические тесты

**Для выполнения нужно знать:** chi-square, Shapiro-Wilk, t-test, Mann-Whitney, Cohen's h, поправка Бонферрони.

Визуализация показала разницу. Теперь нужно убедиться, что эта разница — не случайность.

**Выполни следующие шаги:**

1. **Целевая метрика — No-show rate:** применить chi-square тест по таблице сопряжённости (`pd.crosstab` → `chi2_contingency`). Вывести χ², p-value, степени свободы. Вывод: отвергаем H₀?
2. **Выручка (только `«Проведён»`):**
   - Проверить нормальность каждой группы через `scipy.stats.shapiro`. Вывести p-value Шапиро.
   - На основе результата выбрать тест: если обе группы нормальны → t-test; иначе → Mann-Whitney (`scipy.stats.mannwhitneyu`). Обосновать выбор в Markdown.
3. **Доля повторных визитов:** chi-square тест.
4. **Поправка Бонферрони:** у тебя 3 теста. Скорректированный порог значимости: `alpha_bonf = 0.05 / 3 ≈ 0.0167`. Проверь, остаются ли результаты значимыми после поправки.
5. Рассчитай **Cohen's h** для разницы в No-show rate:
   ```python
   from statsmodels.stats.proportion import proportion_effectsize
   h = proportion_effectsize(p_a, p_b)
   ```
   Интерпретируй: |h| < 0.2 — малый, 0.2–0.5 — средний, > 0.5 — большой эффект.
6. Рассчитай **95% ДИ для разницы пропорций**: `(p_A − p_B) ± 1.96 * sqrt(p_A*(1-p_A)/n_A + p_B*(1-p_B)/n_B)`.
